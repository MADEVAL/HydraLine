# Security Policy — Hydraline

| Поле | Значение |
|---|---|
| Базис | `HYDRALINE_SPEC_V3.md` NF-6, R5 (Critical), §5.4 |
| SLA фикса | Security-патч ≤ **48 часов** (patch-версия) + advisory + регрессионный тест |
| Инвариант | `I2`: 0 XSS на 10^6 входов, 0 падений за 60с фаззинга |

---

## 1. Поддерживаемые версии

Security-фиксы выпускаются для последнего мажора и предыдущего мажора (при наличии).
Пред-релизы (0.x) — только последняя минорная линия.

## 2. Как сообщить об уязвимости

**Не открывайте публичный issue для уязвимостей.** Используйте приватный канал
GitHub Security Advisory (репозиторий → *Security* → *Report a vulnerability*).
Укажите: версию, минимальный reproduction, вектор, ожидаемое/фактическое поведение.

**Процесс реагирования:**
1. Подтверждение получения ≤ 48 ч.
2. Триаж и воспроизведение, присвоение severity (CVSS).
3. Патч + регрессионный тест на конкретный вектор.
4. Публикация patch-версии + GitHub Security Advisory.
5. Кредит репортёру (по желанию).

## 3. Модель угроз (threat model)

Hydraline генерирует HTML из пользовательских данных и монтирует Flutter-острова.
Основные доверенные границы и поверхности атаки:

```
[данные разработчика/БД] → DocumentNode → сериализатор → HTML → браузер → острова
        (T1)                  (T2)            (T3)                    (T4)
```

### T1 — Ввод в модель (XSS через текст/атрибуты) · R5 · Critical
- **Угроза:** пользовательский текст (`<script>`, `" onerror=`) попадает в HTML без экранирования.
- **Контрмеры:**
  - `TextNode`/значения атрибутов **всегда** экранируются на сериализации (`S2`).
  - Контекстные функции: `escapeHtmlText` ≠ `escapeHtmlAttribute` (нельзя перепутать).
  - Property/fuzz-набор с OWASP-векторами (`I2`, тег `security`).
- **Проверка:** `dart test --tags security`; CI-гейт джоба 6 (`DEVELOPMENT §9.2`).

### T1b — URL-инъекции (`javascript:`/`data:`/`vbscript:`)
- **Угроза:** `href`/`src` из ввода исполняет скрипт при клике.
- **Контрмеры:** `AnchorNode`/`ImageNode`/`LinkNode` принимают только `SafeUrl`;
  allowlist схем: `http`,`https`,`mailto`,`tel`,относительные (`S1`, `N3`).
  Сконструировать узел с непроверенным URL **невозможно** на уровне типов.

### T2 — `UnsafeHtmlNode` (осознанный обход)
- **Угроза:** сырой HTML без санитайза → XSS.
- **Контрмеры:** единственный путь для raw-HTML; имя содержит «Unsafe»; валидатор
  предупреждает при `sanitizer == null` (`S3`); статический анализ на использование
  без санитайзера. Санитайзер предоставляет разработчик (осознанный opt-in).

### T3 — Транспорт / cloaking (R7)
- **Угроза 3a (cloaking):** соблазн отдавать разный контент боту и юзеру → санкции поисковиков.
- **Контрмеры:** билдер контента **UA-слеп** (`SRV1`, сигнатура без User-Agent);
  UA доступен только транспорту для выбора buffered/chunked; тело идентично
  (`I3`/`A8`). Это архитектурный, а не дисциплинарный запрет.
- **Угроза 3b (CSP):** инлайн-скрипт при проскочившем XSS.
- **Контрмеры:** рекомендованный CSP `script-src 'self' 'wasm-unsafe-eval'`
  (без `unsafe-inline`); helper `Csp.metaTag()`/`Csp.recommendedHeaderValue()`. HTMX и vanilla —
  self-hosted first-party (совместимо с `script-src 'self'`), без внешних CDN.

### T4 — Граница `data-state` / props (клиент)
- **Угроза:** инъекция через `data-state` при десериализации.
- **Контрмеры (`DS1`):** только `JSON.parse` — **никакого** `eval`/`Function`/`DOMParser`.
  Значения атрибута HTML-экранируются на сервере. Типы ограничены JSON-safe (`DS2`).
  Лимит ~10 KB/остров (`DS4`).

### T5 — Секреты в логах (NF-6)
- **Угроза:** структурные логи сервера содержат токены/PII.
- **Контрмеры:** запрет логирования тел запросов с чувствительными полями;
  ревью логирующих мест; тест на отсутствие секретов в выводе.

## 4. Сводка контрмер → проверка

| Угроза | Контрмера | Инвариант | Задача |
|---|---|---|---|
| T1 XSS-текст | контекстное экранирование | S2, I2 | P1-02, P1-21 |
| T1b URL | `SafeUrl` allowlist | S1, N3 | P1-02 |
| T2 raw-HTML | `UnsafeHtmlNode` + предупреждение | S3 | P1-07 |
| T3a cloaking | UA-слепой билдер + идентичность | SRV1, I3 | P2-02,04,12 |
| T3b CSP | рекомендованный заголовок + self-hosted JS | S5 | P1-02, P2-08 |
| T4 data-state | только JSON.parse | DS1 | P1-16, P4-04 |
| T5 логи | запрет секретов | NF-6 | P2-* |

---

## 5. Безопасные значения по умолчанию (принцип P3)

- Дерево `DocumentNode` **без** `UnsafeHtmlNode` гарантированно безопасно.
- Все url-поля — `SafeUrl` (нельзя обойти без явного санитайза).
- CSP-helper из коробки; фаззинг — часть CI, не опция.

*При обнаружении расхождения между этим документом и поведением кода — приоритет у
регрессионного security-теста; документ приводится в соответствие.*
